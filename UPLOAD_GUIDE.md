# 📤 How to Upload This Project to GitHub (Web Interface)

A simple, beginner-friendly guide — no command line needed. Just your browser.

---

## Before you start
- Have a free **GitHub account** (sign up at https://github.com if you don't).
- Download and **unzip** this project folder so you can see all the files
  (`README.md`, `LICENSE`, the `data`, `dashboard`, and `screenshots` folders, etc.).

---

## Step 1 — Create a new repository
1. Log in to GitHub.
2. Click the **➕ (plus icon)** at the top-right → **New repository**.
3. **Repository name:** `haymedics-africa-health-dashboard`
4. **Description:** `Interactive Excel dashboard of African health indicators (2000–2022) with choropleth maps.`
5. Choose **Public** (recruiters need to see it).
6. ✅ Leave “Add a README” **unchecked** — you already have one.
7. Click **Create repository**.

---

## Step 2 — Upload the files
1. On your new empty repo page, click **“uploading an existing file”** (the blue link),
   or go to **Add file → Upload files**.
2. Open the unzipped project folder on your computer.
3. **Select everything inside it** (README.md, LICENSE, .gitignore, UPLOAD_GUIDE.md,
   and the `dashboard`, `data`, `screenshots` folders) and **drag it all** into the
   upload box on GitHub.
   - 💡 Drag the **contents**, not the outer folder — so `README.md` lands at the top
     level. This makes your README and its images display correctly.
4. Wait for every file to finish uploading (you'll see them listed).

---

## Step 3 — Commit (save) the upload
1. Scroll down to **“Commit changes”**.
2. In the message box type: `Add Africa Health Dashboard project`
3. Click **Commit changes**.

🎉 Done! Refresh the page — your README, the four maps, and all screenshots will
render on the repo home page.

---

## Step 4 — Make it shine for recruiters
1. Click the **⚙️ gear** next to **About** (top-right of the repo).
2. Add a short description and a few **topics**:
   `excel` · `data-analytics` · `dashboard` · `healthcare` · `data-visualization` · `portfolio`
3. Open `README.md` → click the **✏️ pencil** → replace
   *“Add your LinkedIn / portfolio link here”* with your real link → **Commit**.

---

## ✅ Final check
- [ ] README displays with the maps image at the top
- [ ] All four map screenshots load
- [ ] The `.xlsx` file is in the `dashboard` folder
- [ ] The repo is **Public**
- [ ] Your LinkedIn/portfolio link is added

---

### If an image doesn't show
It almost always means a **path or name mismatch**. Check that:
- the `screenshots` folder uploaded (with all its `.png` files), and
- the file names match exactly what the README points to
  (e.g. `screenshots/maps_grid.png`). GitHub is **case-sensitive**.

---

> Tip: copy your repo's link (e.g. `github.com/yourname/haymedics-africa-health-dashboard`)
> straight into your CV and LinkedIn “Featured” section.
